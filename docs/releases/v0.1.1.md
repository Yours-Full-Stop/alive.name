# alive.name v0.1.1

*your name, alive.*

A small patch release. No new commands, no behaviour you have to relearn: just a
friendlier `reclaim` gate and a fix for running the tool from Docker on Windows.
It never pushes and never commits for you, exactly as before.

## What changed

- **`reclaim` no longer stops for files it would never touch.** The working-tree
  check now refuses only on uncommitted changes to **tracked** files. A history
  rewrite cannot affect untracked or ignored files (build output, editor folders
  like `.vscode` or `.idea`, anything gitignored), so their presence no longer
  blocks a rewrite. When any are present, `reclaim` lists them first so you can
  see exactly what is being left untouched, and the refusal message now points you
  to the right remedy ("commit or stash them first").

- **Docker on Windows no longer sees a clean repository as dirty.** A repository
  checked out on Windows uses CRLF line endings; mounted into the Linux container,
  git used to report every text file as modified, which made `reclaim` refuse on a
  tree that was actually clean. The image now sets `core.autocrlf=input` so the
  comparison ignores that difference. It never rewrites your files.

## Why the gate change is safe

`git filter-repo` rewrites committed history, not your working directory's loose
files. Untracked and ignored files are simply not part of what a rewrite reads or
writes, so refusing on them protected nothing. `reclaim` still refuses on any
uncommitted change to a tracked file, still requires a verified backup, still runs
as a dry run unless you pass `--apply`, and still never pushes. Listing the
untracked files keeps the decision transparent: the tool tells you what it is
leaving alone rather than silently ignoring it.

## Install

**Prebuilt binary:** download the archive for your platform from the release
assets, verify it against `checksums.txt`, and put `alive` on your `PATH`.
Available for Linux, macOS, and Windows (amd64 and arm64).

**With Go:**

```bash
go install alive.name/cmd/alive@v0.1.1
```

**With Docker** (bundles `git-filter-repo` so `reclaim` works out of the box):

```bash
docker pull ghcr.io/yours-full-stop/alive.name:0.1.1
docker run --rm -it \
  -v "/path/to/working/repo:/repo" \
  -v "/path/to/backups:/backups" \
  ghcr.io/yours-full-stop/alive.name:0.1.1
```

Both mounts are required; the container refuses backup-creating commands without
`/backups` so a backup can never be ephemeral.

## Upgrading

Nothing to migrate from v0.1.0. Replace the binary (or pull the new image) and
carry on. Backups made with v0.1.0 remain valid.
