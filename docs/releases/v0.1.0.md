# alive.name v0.1.0

*your name, alive.*

The first pre-release. alive.name finds an old name buried in a git history and
helps you make it yours again: safely, on your own machine, with a verified
backup before anything is ever touched. **It never pushes and never commits for
you.**

This is an early, tester-facing build. The core is complete and well tested, but
it stays below 1.0 while real people try it on real repositories. Rough edges and
feedback are exactly what this release is for.

## What's in it

- **`alive`** (or `alive guide`): a calm, guided walkthrough that holds your hand
  from finding the name to fixing it, asking at every step.
- **`alive trace`**: a safe, read-only scan that shows every place the old name
  lives, sorted into three plain-language zones (only on your machine, on a remote
  you control, and beyond your reach).
- **`alive mend`**: writes a `.mailmap` so git displays your new name everywhere.
  History is untouched, and it does not commit the file.
- **`alive reclaim`**: the gated history rewrite via `git filter-repo`. Dry run by
  default, requires a verified backup, refuses a dirty tree, and warns about
  signed or already-published history. After a rewrite it prints the publish
  commands for you to run.
- **`alive backup`** (`create`, `list`, `restore`, `rm`, `gc`): verified backups
  kept outside your working tree, with `.aliveignore` support and an all-refs
  bundle.
- **`alive cleanup`**: advice for clearing the old name from your shell history and
  other local caches.

## Safety, by design

- The tool never pushes and never commits on your behalf. It prepares and prints
  the exact commands; you run them.
- Nothing destructive runs until a backup exists and has been verified.
- The no-push guarantee is structural: the one package that runs git has no push
  capability.

## Install

**Prebuilt binary:** download the archive for your platform from the release
assets, verify it against `checksums.txt`, and put `alive` on your `PATH`.
Available for Linux, macOS, and Windows (amd64 and arm64).

**With Go:**

```bash
go install alive.name/cmd/alive@v0.1.0
```

**With Docker** (bundles `git-filter-repo` so `reclaim` works out of the box):

```bash
docker pull ghcr.io/yours-full-stop/alive.name:0.1.0
docker run --rm -it \
  -v "/path/to/working/repo:/repo" \
  -v "/path/to/backups:/backups" \
  ghcr.io/yours-full-stop/alive.name:0.1.0
```

Both mounts are required; the container refuses backup-creating commands without
`/backups` so a backup can never be ephemeral.

## Requirements

- `git` on your `PATH`.
- Python and `git-filter-repo` are needed only for `alive reclaim`
  (`pip install git-filter-repo`). The Docker image already includes them.

## Notes for testers

- **Short or common old names can over-match.** `reclaim` replaces the name in
  file contents literally, so searching for `Sam` would also rewrite the `Sam` in
  `Sample`. `alive` warns you at input, and the default dry run shows exactly what
  would change: **read the dry run before you `--apply`**, and pair a short name
  with an email to narrow it.
- The cases most worth exercising (and reporting on) are listed under "Scenarios
  worth exercising" in [CONTRIBUTING.md](../../CONTRIBUTING.md).
- Please try the guided `alive` on a hard-but-safe example and tell us where the
  wording felt cold, confusing, or scary. Friction is a bug.
